# 可编辑打招呼模板设计

## 目标

让工作台保存一套可编辑的六段式打招呼模板，并在每次 DeepSeek 润色请求中同时发送岗位 JD、模板约束和已选个人素材。模板只控制表达结构，不能绕过现有事实校验。

## 数据与兼容

- 继续使用现有 `GreetingTemplate.body` 字段，不新增重复字段。
- 将默认模板正文替换为六段式结构，版本提升到 2。
- 已经保存的 `template.json` 保持原样，不自动覆盖用户内容；用户可以在话术设置页手动替换并保存。
- `bannedPhrases` 继续作为独立的禁用词列表，不能再冒充“模板正文”。

## 默认模板

```text
BOSS您好，我对您发布的{{jobTitle}}岗位很感兴趣。
{{selfIntro}}
主要工作或项目内容包括：{{projects}}。
我的岗位匹配优势包括：{{matchedRequirements}}。
相关技能包括：{{skills}}。
详细情况您可以看下我的简历，期待您的回复！
```

这是可直接发送的回退正文，不包含“请选择三项”等操作说明。DeepSeek 通过独立提示词读取完整 JD，对已有素材进行筛选、排序和自然化表达。

## 生成流程

1. 本地渲染器根据模板、岗位和个人素材生成基础草稿。
2. DeepSeek 润色请求携带完整 `template.body`、语气、长度约束、岗位标题、公司、JD、基础草稿和已选个人素材。
3. 模型只能调整表达和素材顺序，不得新增未提供的公司、年限、技能、职责或成果。
4. 返回内容继续经过素材 ID 白名单、未知事实检测、禁用词和长度校验；失败时回退到本地草稿。

## 界面

- “话术设置”页面增加真正的“话术结构模板”多行编辑框，绑定 `template.body`。
- 单独保留“禁用词”编辑框，绑定 `template.bannedPhrases`。
- 页面提示模板会随每次 DeepSeek 生成发送，但 API Key 和模型配置仍只从环境变量读取。

## 验证

- 测试模板正文可以独立编辑和保存，禁用词不受影响。
- 测试 DeepSeek 润色提示词包含模板正文和完整 JD。
- 测试默认模板为六段式且包含岗位占位符。
- 运行 Web 全量测试、类型检查和构建。
